# SIDIAN — REQUIREMENTS & TRACEABILITY MATRIX

**Version : 1.2 — revue substantielle**  
**Statut : VALIDÉE AU NIVEAU DOCUMENTAIRE**  
**Sources :** documents 04 à 09 de SIDIAN Specification v1.0  
**Objet :** relier chaque exigence normative à un ou plusieurs cas d’évaluation qui contrôlent réellement son contenu.

> Cette matrice prouve la cohérence de la couverture documentaire. Elle ne prouve pas encore que les tests ont été implémentés ni exécutés sur le code.

# 1 — Méthode de validation

La revue a été effectuée exigence par exigence, puis cas d’évaluation par cas d’évaluation.

Pour chaque ligne, les contrôles suivants ont été appliqués :

- correspondance du texte avec la source normative indiquée ;
- pertinence réelle du scénario d’évaluation ;
- suffisance du résultat attendu ;
- distinction entre couverture directe et composite ;
- vérification des identifiants ;
- recherche des tests sans exigence ni risque documenté ;
- création de nouveaux cas lorsque le catalogue v1.1 ne couvrait pas substantiellement une exigence.

Valeurs utilisées :

- `direct` : un cas contrôle directement l’ensemble de l’exigence ;
- `composite` : plusieurs cas complémentaires sont nécessaires ;
- `covered` : la couverture documentaire est substantielle ;
- `transversal justified` : le cas contrôle un risque global sans correspondre à une exigence atomique unique.

# 5 — Lacunes fermées pendant la revue

La revue a identifié seize contrôles absents du catalogue v1.1. Ils ont été ajoutés à `09_AGENT_EVALUATIONS.md` v1.2 :

- architecture des couches de prompt ;
- distinction des catégories de mémoire ;
- topologie monolithe modulaire EPICU V1 ;
- séparation configuration/données opérationnelles ;
- évaluation complète de la réversibilité ;
- préflight de décision ;
- taxonomie des erreurs outils ;
- interdiction de décision métier dans un outil ;
- indépendance de la mémoire vis-à-vis du LLM ;
- schéma obligatoire d’un souvenir ;
- traitement asynchrone et audité des webhooks ;
- interdiction d’accès direct du LLM ;
- immutabilité des événements d’audit ;
- alerte de dérive de permission ;
- contrat obligatoire du LLM Gateway ;
- préparation opérationnelle aux incidents avant production.
